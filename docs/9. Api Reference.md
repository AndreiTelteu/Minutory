## Introduction
This document provides comprehensive RESTful API documentation for the **meetingai** application. It covers all public endpoints defined in the `web.php` routes file, detailing HTTP methods, URL patterns, request parameters, response schemas, and usage examples. The API supports session-based authentication and includes CSRF protection for state-changing operations. Endpoints are grouped by controller and functionality, with detailed specifications for request and response formats.

## Authentication and Security
All API endpoints require **session-based authentication**. Users must be logged in to access any endpoint. The application uses Laravel's built-in session management and CSRF protection via middleware.

CSRF tokens are required for all `POST`, `PUT`, and `DELETE` requests. These should be included in the request headers:

```
X-CSRF-TOKEN: {csrf-token}
```


Authentication is enforced through Laravel's session guard and is automatically handled by the framework for all routes.

## Meeting Endpoints

### POST /meetings (Upload Meeting)
Uploads a new meeting video file and creates a meeting record.

**HTTP Method**: `POST`  
**URL**: `/meetings`  
**Authentication**: Required  
**CSRF Protection**: Required  

#### Request Parameters
| Field | Type | Required | Description |
|------|------|----------|-------------|
| `title` | string | Yes | Meeting title (max 255 characters) |
| `client_id` | integer | Yes | ID of associated client |
| `video` | file | Yes | Video file (MP4, MOV, AVI, WebM; 1MB–500MB) |

#### Request Body Example (multipart/form-data)

```
title=Team Sync Meeting
client_id=1
video=@meeting_video.mp4
```


#### Response Schema (JSON)
On success, returns a redirect to `/meetings` with a success message in session.

**Success Response (302 Redirect)**:

```
Location: /meetings
Set-Cookie: XSRF-TOKEN=...
```


**Error Responses**:
- `400 Bad Request`: Validation errors (e.g., invalid file type, missing fields)
- `500 Internal Server Error`: Upload or processing failure

#### Sample curl Request

```bash
curl -X POST \
  http://localhost/meetings \
  -H "X-CSRF-TOKEN: abc123" \
  -F "title=Project Kickoff" \
  -F "client_id=2" \
  -F "video=@project_kickoff.mp4" \
  -b "XSRF-TOKEN=abc123; laravel_session=xyz789"
```

### GET /meetings (List Meetings)
Retrieves a paginated list of meetings with optional filtering and sorting.

**HTTP Method**: `GET`  
**URL**: `/meetings`  
**Authentication**: Required  

#### Query Parameters
| Parameter | Type | Required | Description |
|---------|------|----------|-------------|
| `client_id` | integer | No | Filter by client ID |
| `status` | string | No | Filter by status (`pending`, `processing`, `completed`, `failed`) |
| `date_from` | date (Y-m-d) | No | Filter meetings uploaded on or after this date |
| `date_to` | date (Y-m-d) | No | Filter meetings uploaded on or before this date |
| `sort` | string | No | Sort by `uploaded_at`, `title`, `status`, `duration`, `client` |
| `direction` | string | No | Sort direction: `asc` or `desc` (default: `desc`) |

#### Response Schema (JSON)
Returns Inertia response with structured data.


```json
{
  "meetings": {
    "data": [
      {
        "id": 1,
        "title": "Team Sync",
        "status": "completed",
        "duration": 1800,
        "uploaded_at": "2025-08-10T10:00:00.000000Z",
        "client": {
          "id": 1,
          "name": "Acme Corp"
        }
      }
    ],
    "links": {
      "first": "/meetings?page=1",
      "last": "/meetings?page=5",
      "prev": null,
      "next": "/meetings?page=2"
    }
  },
  "clients": [
    {"id": 1, "name": "Acme Corp"},
    {"id": 2, "name": "Beta Inc"}
  ],
  "filters": {
    "client_id": null,
    "status": null,
    "date_from": null,
    "date_to": null,
    "sort": "uploaded_at",
    "direction": "desc"
  }
}
```


#### Sample curl Request

```bash
curl -G \
  http://localhost/meetings \
  -d client_id=1 \
  -d status=completed \
  -d sort=title \
  -d direction=asc \
  -b "laravel_session=xyz789"
```

### GET /meetings/{meeting}/status (Real-time Status)
Provides real-time processing status for a meeting.

**HTTP Method**: `GET`


- `200 OK`: Status retrieved successfully
- `500 Internal Server Error`: Failed to retrieve status


## AI Agent Endpoints

### POST /ai/chat (Conversational Search)
Engages in a conversation with the AI agent to search meeting content.

**HTTP Method**: `POST`  
**URL**: `/ai/chat`  
**Authentication**: Required  
**CSRF Protection**: Required  

#### Request Parameters
| Field | Type | Required | Description |
|------|------|----------|-------------|
| `message` | string | Yes | User message (max 1000 characters) |
| `conversation_history` | array | No | Array of `{role, content}` objects (max 50 entries) |

#### Request Body Example

```json
{
  "message": "What did Alice say about the deadline?",
  "conversation_history": [
    {
      "role": "user",
      "content": "Summarize the last meeting"
    },
    {
      "role": "assistant",
      "content": "The meeting discussed project timelines..."
    }
  ]
}
```


#### Response Schema

```json
{
  "success": true,
  "response": "Alice mentioned the deadline is next Friday at 5 PM.",
  "tool_calls": [
    {
      "name": "search_meetings",
      "arguments": {
        "query": "deadline",
        "speaker": "Alice"
      }
    }
  ]
}
```


**Error Responses**:
- `422 Unprocessable Entity`: Validation error
- `429 Too Many Requests`: Rate limit exceeded (10 requests/minute)
- `408 Request Timeout`: AI service timeout
- `503 Service Unavailable`: Network error

#### Sample curl Request

```bash
curl -X POST \
  http://localhost/ai/chat \
  -H "X-CSRF-TOKEN: abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Find discussions about budget",
    "conversation_history": []
  }' \
  -b "XSRF-TOKEN=abc123; laravel_session=xyz789"
```

### POST /ai/search (Direct Search)
Performs a direct search across meeting transcriptions.

**HTTP Method**: `POST`  
**URL**: `/ai/search`  
**Authentication**: Required  
**CSRF Protection**: Required  

#### Request Parameters
| Field | Type | Required | Description |
|------|------|----------|-------------|
| `query` | string | Yes | Search query (max 500 characters) |
| `client_id` | integer | No | Filter by client ID |
| `speaker` | string | No | Filter by speaker name |
| `limit` | integer | No | Maximum results (1–50, default: 10) |

#### Request Body Example

```json
{
  "query": "project timeline",
  "client_id": 1,
  "speaker": "Bob",
  "limit": 5
}
```


#### Response Schema

```json
{
  "success": true,
  "data": {
    "results": [
      {
        "meeting_id": 1,
        "meeting_title": "Team Sync",
        "client_name": "Acme Corp",
        "speaker": "Bob",
        "text": "The **project timeline** has been adjusted to Q4.",
        "timestamp": 125.5,
        "formatted_timestamp": "02:05",
        "confidence": 0.92,
        "meeting_url": "http://localhost/meetings/1?t=125.5"
      }
    ],
    "total_found": 1,
    "search_query": "project timeline",
    "client_filter": 1,
    "speaker_filter": "Bob"
  }
}
```


**Error Responses**:
- `400 Bad Request`: Empty query
- `500 Internal Server Error`: Search failure

#### Sample curl Request

```bash
curl -X POST \
  http://localhost/ai/search \
  -H "X-CSRF-TOKEN: abc123" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "budget approval",
    "limit": 3
  }' \
  -b "XSRF-TOKEN=abc123; laravel_session=xyz789"
```

## Error Handling
The API returns standardized error responses for failed requests.

### Common Error Status Codes
| Status Code | Meaning | Description |
|------------|--------|-------------|
| `400` | Bad Request | Invalid request parameters |
| `401` | Unauthorized | Authentication required |
| `404` | Not Found | Resource not found |
| `422` | Unprocessable Entity | Validation errors |
| `429` | Too Many Requests | Rate limit exceeded |
| `500` | Internal Server Error | Server-side error |

### Error Response Format

```json
{
  "success": false,
  "error": "Descriptive error message"
}
```


For validation errors, Laravel returns a redirect with error messages in the session.

## Rate Limiting and Usage Constraints
- **AI Chat Endpoint**: Limited to 10 requests per minute per IP address.
- **File Uploads**: Maximum 500MB per video file.
- **Search History**: Maximum 50 messages in conversation history.
- **Search Results**: Maximum 50 results per request.

Rate limiting is implemented using Laravel's cache system with a 1-minute window.
