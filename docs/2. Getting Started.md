## Prerequisites

Before setting up the meetingai application, ensure your system meets the following requirements:

- **PHP 8.2+** (as specified in `composer.json`)
- **Node.js 18+** (compatible with Vite and Vue 3)
- **Docker** (to run the transcription microservice in a container)
- **Composer** (PHP dependency manager)

These tools are essential for running the Laravel backend, Vue 3 frontend, and the Python-based transcription microservice.

**Section sources**
- [composer.json](file://composer.json#L10)
- [package.json](file://package.json#L25)

## Clone the Repository

Begin by cloning the repository to your local machine:


```bash
git clone https://github.com/your-organization/meetingai.git
cd meetingai
```


This creates a local copy of the project with the full directory structure, including Laravel backend, Vue 3 frontend, and the transcription microservice.

## Install PHP Dependencies

Use Composer to install all required PHP packages:


```bash
composer install
```


This command reads `composer.json` and installs Laravel 12, Inertia.js integration, Prism AI agent, and other dependencies. It also sets up the autoloader for PSR-4 compliant namespaces.

**Expected Output Example:**

```
Loading composer repositories with package information
Installing dependencies from lock file
Package operations: 87 installs, 0 updates, 0 removals
...
Generating autoload files
```

## Configure Environment Variables

Create a `.env` file from the example template and configure key settings:


```bash
cp .env.example .env
```


If no `.env.example` exists, create `.env` manually with the following essential configurations:

### Database Settings

```env
DB_CONNECTION=sqlite
# Or for MySQL:
# DB_CONNECTION=mysql
# DB_HOST=127.0.0.1
# DB_PORT=3306
# DB_DATABASE=meetingai
# DB_USERNAME=root
# DB_PASSWORD=
```


The application supports SQLite (default) or MySQL. For development, SQLite is simpler.

### Queue Settings

```env
QUEUE_CONNECTION=database
```


This uses the database queue driver, which stores jobs in the `jobs` table. No Redis or external queue is required for local development.

### OpenRouter API Settings

```env
OPENROUTER_API_KEY=your_openrouter_api_key_here
```


This key enables AI-powered search and transcription analysis via the Prism agent. You can obtain a key from [OpenRouter.ai](https://openrouter.ai).

### Application Key
Generate a unique application key:


```bash
php artisan key:generate
```


This sets `APP_KEY` in `.env`, required for session encryption and security.

## Run Database Migrations

Set up the database schema by running migrations:


```bash
php artisan migrate
```


This creates tables for users, clients, meetings, transcriptions, and queue jobs as defined in the `database/migrations` directory.

To populate sample data:


```bash
php artisan db:seed
```


This runs seeders to create test clients and meetings.

## Start Development Servers

Launch all development services concurrently:


```bash
composer dev
```


This single command starts:
- Laravel development server (`php artisan serve`)
- Queue worker (`php artisan queue:listen`) for background transcription jobs
- Vite development server (`npm run dev`) for hot-reloading frontend assets

**Alternative: Manual Commands**

```bash
php artisan serve
php artisan queue:listen --tries=1 --timeout=3600 --memory=512
npm run dev
```


**Expected Output:**

```
[server] Laravel development server started...
[queue] Processing jobs on database:default
[vite] Vite dev server running at http://localhost:5173
```

## Run Transcription Microservice

The transcription service runs in Docker using WhisperX and pyannote.audio for speaker diarization.

Build and run the container:


```bash
cd transcribe-microservice
docker build -t meetingai-transcribe .
docker run -d \
  -v ./uploads:/scriberr/uploads \
  -v ./models:/scriberr/models \
  -p 8001:8001 \
  meetingai-transcribe
```


This starts a Python microservice that processes uploaded videos into transcriptions with speaker identification.

**Dockerfile Notes:**
- Uses Python 3.11-slim base image
- Installs `ffmpeg`, `whisperx`, `faster-whisper`, and `pyannote-audio`
- Exposes port 8001 for API access
- Uses tini for proper signal handling

## Troubleshooting Common Issues

### Queue Worker Not Processing Jobs
**Symptom**: Upload completes but transcription status remains "pending".

**Solution**: Ensure `php artisan queue:listen` is running. Restart it if needed:

```bash
php artisan queue:restart
```


Verify the queue connection in `.env` matches your database setup.

### Inertia.js Routing Errors
**Symptom**: Blank page or 404 after navigation.

**Solution**: Clear Laravel and Vite caches:

```bash
php artisan view:clear
php artisan route:clear
npm run dev
```


Ensure `APP_URL` in `.env` matches the server address (e.g., `http://localhost:8000`).

### Microservice Connectivity Issues
**Symptom**: Transcription jobs fail with connection errors.

**Solution**: Verify Docker container is running:

```bash
docker ps
```


Check logs:

```bash
docker logs <container_id>
```


Ensure the Laravel app can reach `http://localhost:8001` and that the service is bound correctly.

### Vite Assets Not Loading
**Symptom**: CSS/JS not loading, console shows 404 for `/resources/...`.

**Solution**: Ensure `npm run dev` is running and Vite server is active. Check `vite.config.ts` for correct input paths.

## Basic Usage Walkthrough

### Step 1: Upload a Test Meeting
1. Navigate to **Meetings > Create**
2. Select a client
3. Upload a video file (MP4, WebM, etc.)
4. Submit the form

**Expected Outcome**: Meeting appears in list with status "Processing".

### Step 2: View Processing Status
- Check the meeting list for real-time status updates
- The queue worker will process the job and update status to "Completed"
- Transcription data is stored in the `transcriptions` table

### Step 3: Perform AI Search Query
1. Go to **AI Chat**
2. Ask a question like: *"Find all meetings where budget was discussed"*
3. The AI agent uses `MeetingSearchTool` to query transcriptions

**Expected Outcome**: Results with timestamps and deep links to relevant meeting segments.

This completes the local setup and basic usage of the meetingai application.
